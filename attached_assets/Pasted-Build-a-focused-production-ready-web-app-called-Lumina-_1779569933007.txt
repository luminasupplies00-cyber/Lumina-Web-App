Build a focused, production-ready web app called "Lumina HQ"
for Lumina Supplies — a B2B laboratory and scientific supplies
company in Riyadh, Saudi Arabia.

I am the business owner. My single biggest daily bottleneck is
processing RFQ emails (Requests for Quotation) from hospitals,
labs, and research institutions.

My current manual process per RFQ:
1. Open email in Zoho Mail
2. Read and understand what products are needed
3. Extract the product list manually
4. Email suppliers to get pricing
5. Wait for supplier replies
6. Compare supplier prices manually
7. Calculate selling price with markup
8. Write and send customer quotation email
9. Follow up if no customer response after a few days

This takes 30-60 minutes per RFQ.
I receive multiple RFQs per day.
Goal: reduce this to under 10 minutes per RFQ.

## PART 1: ZOHO MAIL CONNECTION

Implement real Zoho Mail OAuth 2.0 — no mock emails.

OAuth flow:
- Register as Server-Based Application in Zoho API Console
- Authorize at: {accounts_domain}/oauth/v2/auth
- Callback route: /api/auth/zoho/callback
- Exchange code for access_token + refresh_token
- Store tokens encrypted in DB
- Auto-refresh when token expires

Config fields (in Settings/Integrations page):
- ZOHO_CLIENT_ID
- ZOHO_CLIENT_SECRET
- ZOHO_REDIRECT_URI
- ZOHO_ACCOUNTS_DOMAIN (default: accounts.zoho.com)

After connection:
- Fetch Zoho Mail account ID
- Sync inbox threads
- Run AI triage on each thread

Authorization header for all Zoho Mail API calls:
Authorization: Zoho-oauthtoken {access_token}
NOT Bearer.

## PART 2: AI EMAIL TRIAGE

On every sync, classify each email thread using
Perplexity sonar-pro.

Classify as one of:
- RFQ (customer requesting quotation)
- Supplier Reply (supplier responding to our inquiry)
- Customer Follow-up (customer asking about sent quote)
- PO / Invoice (purchase order or invoice related)
- Internal
- Spam / Newsletter
- General

For RFQ emails, also extract:
- customer name
- customer company
- products mentioned (name, cat no, brand, quantity)
- any deadline mentioned
- urgency signals

Auto-create an RFQ record for every email classified as RFQ.

Show classification badge on every email in the inbox view.

## PART 3: RFQ PIPELINE PAGE

Main page of the app at /rfq

Show RFQ records grouped by stage in a vertical list.

Stages:
1. NEW — received, not yet processed
2. SOURCING — sent to suppliers, awaiting quotes
3. COMPARING — supplier quotes received
4. QUOTE READY — customer quote prepared
5. QUOTE SENT — awaiting customer response
6. FOLLOW UP — no response after 3+ days
7. WON
8. LOST / NO RESPONSE

Each RFQ card shows:
- customer name + company
- email subject
- date received
- days in current stage (red if > 2 days)
- number of products
- estimated value (SAR) if known
- AI suggested next action (one line)
- stage badge
- action buttons (stage-specific)

---

## PART 4: AI PRODUCT EXTRACTION

Button: "Extract Products" on NEW stage cards.

Call Perplexity sonar-pro with email body.
Extract structured product list:

Per product:
- product_name
- catalogue_number (if mentioned)
- brand (if mentioned)
- quantity
- specifications

Return as JSON.
Show in expandable product table on the card.
Store in rfq_products table.

If extraction fails:
- Show clean error message
- Show "Open in Zoho Mail" link

max_tokens: 800
