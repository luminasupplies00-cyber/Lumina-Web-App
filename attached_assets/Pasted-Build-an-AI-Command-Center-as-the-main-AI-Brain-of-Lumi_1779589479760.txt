Build an AI Command Center as the main AI Brain of
Lumina HQ. This is a persistent, trainable AI assistant
that understands Lumina Supplies business context and
can be commanded to do anything within the app.

---

## WHAT IT IS

A dedicated AI Brain page and a floating command bar
accessible from every page in the app.

It is NOT a chatbot.
It is NOT a generic assistant.
It is a business-specific AI operator that knows
Lumina Supplies, knows your workflow, knows your
suppliers, knows your customers, and executes real
actions inside the app.

## PART 1: FLOATING COMMAND BAR

Available on every page via:
- Keyboard shortcut: CMD+K or CTRL+K
- Floating button: bottom right corner of every page
  Icon: brain or command symbol
  Label: "AI Command"

When opened:
- Full-width modal or slide-up panel
- Large text input at top
- Recent commands below
- Suggested commands below that

User types a natural language command and presses Enter.
AI interprets and executes or responds.

Examples of commands it must handle:

DRAFT COMMANDS:
"Draft a supplier email for the KFSH RFQ"
"Write a follow-up for the quote sent to RAMSys 4 days ago"
"Draft a reply to the Merck pricing email"
"Write a professional apology for the delayed quotation"
"Draft a supplier RFQ for centrifuge tubes, 50ml, qty 500"

FIND COMMANDS:
"Show me all unanswered RFQs from this week"
"Which suppliers have not responded in 48 hours"
"Find the quote I sent to King Fahad Hospital"
"Show overdue tasks assigned to Asha"
"Which RFQs are stuck in sourcing"

ACTION COMMANDS:
"Move the KFSH RFQ to comparing stage"
"Mark the Glassco quote as responded"
"Assign the SFDA task to Eman"
"Create a ClickUp task for SFDA registration follow-up"
"Snooze the newsletter emails for 7 days"

ANALYSIS COMMANDS:
"What is the status of my pipeline today"
"Which customers have sent RFQs this month"
"How many quotes are awaiting customer response"
"What should I focus on today"
"Summarize this week's supplier activity"

TRAINING COMMANDS:
"Remember: always address Merck as Merck KGaA"
"Remember: our payment terms are Net 30"
"Remember: for glassware always contact Glassco first"
"Remember: Asha handles all procurement emails"
"Remember: our standard markup is 35%"
"Set default follow-up time to 3 days"

## PART 4: COMMAND EXECUTION ENGINE

When a command is received, the AI must:

1. UNDERSTAND the intent:
   - Is it a DRAFT request?
   - Is it a FIND/QUERY request?
   - Is it an ACTION request?
   - Is it a TRAINING/MEMORY update?
   - Is it an ANALYSIS request?

2. GATHER context:
   - Query the database for relevant data
   - If command mentions an RFQ, supplier, or customer
     by name, look them up first

3. EXECUTE:
   - DRAFT → generate and show in modal
   - FIND → query DB and show results
   - ACTION → execute with confirmation
   - TRAINING → save to AI Brain memory
   - ANALYSIS → query DB and generate AI summary

4. RESPOND with:
   - Result or draft
   - Source items referenced
   - Actions taken or available
   - Follow-up suggestions

Command response format in the UI:
┌─────────────────────────────────────────────┐
│ You: "Draft a follow-up for KFSH quote"     │
├─────────────────────────────────────────────┤
│ AI Brain:                                   │
│                                             │
│ Found: Quote sent to KFSH on May 21         │
│ Days since: 3 days (follow-up threshold)    │
│                                             │
│ Draft:                                      │
│ ┌─────────────────────────────────────────┐ │
│ │ Subject: Follow-up: Quotation QT-000253 │ │
│ │ [editable draft body here...]           │ │
│ └─────────────────────────────────────────┘ │
│                                             │
│ [Copy] [Send via Zoho Mail] [Edit]          │
│                                             │
│ Also: 2 other quotes awaiting response      │
│ [Show all] [Draft follow-ups for all]       │
└─────────────────────────────────────────────┘

## PART 5: TRAINING COMMANDS

When user says "Remember: ..." the AI must:

1. Parse the instruction
2. Identify which memory category it belongs to:
   - Email style → templates
   - Supplier rule → supplier preferences
   - Customer rule → customer rules
   - Team rule → team assignments
   - Business setting → company profile
3. Save to the correct section in ai_brain_memory table
4. Confirm: "Got it. I'll remember that [supplier X]
   should be contacted first for [category Y]."

Memory is persistent across sessions.
Memory is used in every AI call via buildAIContext().
