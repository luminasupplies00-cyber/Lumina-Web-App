## ZOHO MAIL SEND API

Endpoint:
POST https://mail.zoho.com/api/accounts/{accountId}/messages

Headers:
Authorization: Zoho-oauthtoken {access_token}
Content-Type: application/json

Body for plain text + HTML email:
{
  "fromAddress": "faraz@lumina-supplies.com",
  "toAddress": "supplier@example.com",
  "ccAddress": "",
  "bccAddress": "supplier2@example.com,supplier3@example.com",
  "subject": "RFQ — Laboratory Products — Lumina Supplies",
  "content": "HTML email body here",
  "mailFormat": "html",
  "attachments": []
}

For emails with Excel attachment:
- First upload the file to Zoho Mail
- Then reference attachment ID in the send call

Zoho Mail attachment upload:
POST https://mail.zoho.com/api/accounts/{accountId}/messages/attachments
Headers:
  Authorization: Zoho-oauthtoken {access_token}
  Content-Type: multipart/form-data
Body: form-data with file field

Returns: { attachmentId: "xxx" }

Then include in send body:
"attachments": [{ "attachmentId": "xxx" }]

## OAUTH SCOPE CHECK

Before showing Send button, verify the connected
Zoho Mail account has send permission.

Required scope: ZohoMail.messages.ALL or
ZohoMail.messages.CREATE

Check stored token scopes when rendering the modal.

If scope missing:
- Hide "Send via Zoho Mail" button
- Show: "Reconnect Zoho Mail to enable sending"
- Show "Reconnect" button that re-runs OAuth with
  correct scopes including send permission
- Keep "Copy to clipboard" as fallback

If scope present:
- Show "Send via Zoho Mail" button (primary action)
- Keep "Copy" as secondary option

## ERROR HANDLING

Handle these Zoho Mail send errors:

401 — Token expired → auto refresh and retry once
403 — Insufficient scope → show reconnect prompt
429 — Rate limit → wait 2 seconds and retry once
500 — Zoho server error → show error, offer retry
Network error → show error, offer retry

Never silently fail a send.
Always show the user if an email failed to send.
