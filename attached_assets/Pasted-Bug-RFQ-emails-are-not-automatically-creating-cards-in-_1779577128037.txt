THE FIX (implement after diagnosis):

In the email sync flow, after every email is classified,
add this logic:

if (classification === 'RFQ') {
  1. Check if an rfq_record already exists for this 
     thread_id to avoid duplicates
  2. If not exists, create rfq_record with:
     - email_thread_id = thread.id
     - source_email_subject = thread.subject
     - customer_name = rfq_data.customer_name or null
     - customer_company = rfq_data.customer_company or null
     - customer_email = thread.sender_email
     - stage = 'NEW'
     - estimated_value = null
     - created_at = now()
     - stage_updated_at = now()
  3. Log: "Created RFQ record for thread: {subject}"
}
