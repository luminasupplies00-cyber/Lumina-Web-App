### ADDITION 1: ATTACHMENT HANDLING

Most RFQs from Saudi hospitals and labs arrive as PDF or Excel
attachments, not plain email text.

Add attachment parsing to the email sync and triage flow:

For PDF attachments:
- Extract text from PDF using a PDF parsing library
- Pass extracted text to Claude for product extraction
- Same structured output as email body extraction

For Excel (.xlsx / .xls) attachments:
- Parse spreadsheet rows and columns
- Identify product columns: name, cat no, brand, qty, specs
- Map to rfq_products schema
- Show parsed rows in extraction review UI

For images (JPG, PNG — forwarded WhatsApp screenshots etc):
- Use Claude's vision capability to read the image
- Extract product information as best as possible
- Flag as "needs review" with lower confidence score

Add a dedicated EXTRACTION REVIEW screen:
- Shows AI-extracted products in an editable table
- User can:
  - Edit any field inline
  - Delete a row
  - Add a missing row manually
  - Confirm and advance to next stage
- Must be completed before RFQ can advance from NEW stage
- This is a financial risk control, not just a UX nicety

Update database:
- Add attachment_parsed boolean to email_threads
- Add attachment_type field to rfq_products
  (body | pdf | excel | image)
- Add extraction_confidence field to rfq_products
  (high | medium | low | manual)
- Add extraction_reviewed boolean to rfq_records
- Add extraction_reviewed_at timestamp

### ADDITION 7: LANDED COST BUFFER

The markup calculator must account for freight and customs.

Update the comparison table markup calculator:

Formula:
Landed Cost = Supplier Price × (1 + landed_cost_buffer%)
Selling Price = Landed Cost × (1 + markup%)

Add to comparison table UI:
- Landed cost buffer % field (default 8%, editable per RFQ)
- Shows: Supplier Price → Landed Cost → Selling Price
- Tooltip explaining what the buffer covers

This prevents accidentally quoting below actual cost.

### ADDITION 8: STUCK RFQ ALERTS

Add automatic detection of stuck RFQs.

Business rules:
- NEW stage > 4 hours during business hours → flag as stuck
- SOURCING stage > 48 hours → flag as stuck
- COMPARING stage > 24 hours → flag as stuck
- QUOTE READY stage > 4 hours → flag as stuck
- QUOTE SENT stage > 3 days → flag for follow-up
- FOLLOW UP stage > 5 days without action → escalate

Show stuck flags:
- Red indicator on RFQ card
- Count in metrics strip ("X stuck")
- Sorted to top within their stage section
